# 廖雪峰写作框架核心规律

## 文章整体结构

### 标准三段式结构
1. **引入段** - 提出问题/场景，激发兴趣
2. **展开段** - 层层递进，原理→例子→实践
3. **总结段** - 核心要点回顾，强调实践价值

### 章节组织规则
- 每个大节解决一个核心问题
- 小节控制在3-7个之间
- 使用数字编号建立层次感（1.1, 1.2, 1.3）
- 每小节长度控制在200-500字

## 逻辑展开模式

### 模式1：问题驱动型
```
为什么需要X？
→ 没有X会遇到什么问题（痛点）
→ X如何解决这个问题（方案）
→ 用一个例子说明（实例）
→ 小结：所以X很重要
```

### 模式2：对比递进型
```
传统方法A是这样做的
→ 但是A有什么局限
→ 新方法B改进了什么
→ 用对比例子说明差异
→ 小结：什么时候用A，什么时候用B
```

### 模式3：层层深入型
```
先看最简单的情况
→ 理解了吗？我们加一点复杂度
→ 再看一个更实际的场景
→ 最后看一个完整的例子
→ 小结：从简单到复杂的规律
```

## 句式节奏控制

### 核心原则
- **短句为主**：单句不超过25字
- **问答搭配**：每2-3段提出一个问题
- **强调重复**：关键概念出现3次以上

### 典型句式组合
1. 提问句（引起思考）
2. 陈述句（给出答案）
3. 举例句（具体说明）
4. 对比句（深化理解）
5. 总结句（强化记忆）

示例：
```
什么是变量？
变量就是用来存储数据的容器。
就像一个盒子，你可以往里面放东西，也可以取出来。
和常量不同，变量的值可以改变。
记住：变量=可变的数据容器。
```

## 解释技术原则

### 通俗化策略
1. **生活类比**：用日常事物类比技术概念
   - 变量→盒子
   - 函数→工具箱
   - 循环→重复做某事
   - 数组→排队的人

2. **视觉化描述**：用"看到""想象"等词
   - "我们可以把它想象成..."
   - "如果你看控制台，会发现..."
   - "注意观察这里的变化..."

3. **分步拆解**：复杂概念分3-5步
   - 第一步：最简单的情况
   - 第二步：加一个条件
   - 第三步：完整场景
   - 避免一次性抛出完整复杂度

### 术语处理
- 首次出现：中文解释 + 英文原词
- 再次出现：只用中文
- 重要术语：加粗或代码格式
- 示例："我们把函数（function）理解为一个工具箱"

## 例子编写规范

### 例子的四要素
1. **场景具体**：不说"处理数据"，说"统计学生成绩"
2. **代码简短**：核心代码不超过10行
3. **结果可见**：必须有输出/效果展示
4. **循序渐进**：从错误示例→正确示例

### 例子模板
```
假设我们要[具体任务]。

错误的做法：
[代码 - 展示常见错误]

运行结果：
[错误输出]

正确的做法：
[代码 - 展示正确方法]

运行结果：
[正确输出]

注意到区别了吗？关键在于[核心差异]。
```

## 练习设计原则

### 练习特征
- 紧扣本节核心知识点
- 难度略低于讲解内容
- 有明确的验证标准
- 用"试试看""动手练习"等鼓励性语言

### 练习模板
```
现在轮到你了！

**练习**：[任务描述]

提示：
- 第一步：[具体指导]
- 第二步：[具体指导]
- 如果遇到困难，回顾一下[相关知识点]

完成后，运行代码，如果看到[预期结果]，说明你做对了！
```

## 过渡衔接技巧

### 段落过渡词库
- **递进**："接下来"、"更进一步"、"我们再看"
- **转折**："但是"、"然而"、"注意"
- **因果**："所以"、"因此"、"这意味着"
- **总结**："综上所述"、"总的来说"、"记住"
- **示例**："比如"、"举个例子"、"假设"

### 章节衔接句式
- "在[上一节]中，我们学习了[X]。现在我们来看[Y]。"
- "掌握了[X]之后，你可能会问：[问题]？"
- "理解了[概念]，下一步就是[实践]。"
- "[X]很重要，但在实际使用中，我们还需要注意[Y]。"

## 语气与态度

### 标志性用语
- **鼓励性**："很简单"、"不用担心"、"只需要"
- **亲和性**："我们"、"你会发现"、"试试看"
- **强调性**："注意"、"重点"、"关键"、"一定要"
- **降低难度**："其实就是"、"说白了"、"简单来说"

### 禁忌用语
- 避免：专业术语堆砌
- 避免：假设读者有背景知识
- 避免："显然"、"众所周知"等假定性词汇
- 避免：长难句、从句套从句

## 标题命名规则

### 标题特征
- 问题式："什么是X？"、"为什么需要X？"、"如何使用X？"
- 动作式："安装X"、"配置Y"、"运行第一个程序"
- 概念式："变量与常量"、"循环与条件"
- 场景式："处理用户输入"、"读写文件"

### 标题层级
```
# 大章节 - 一个主题
## 核心概念 - 一个大问题
### 具体知识点 - 一个小问题
#### 细节说明 - 特殊情况/注意事项
```

## 代码呈现规范

### 代码块要求
- 语言标注：使用正确的语法高亮
- 注释说明：关键行添加行内注释
- 完整性：能直接运行的完整代码
- 长度控制：核心代码<15行，超过则分段

### 代码说明模板
```
看下面的代码：

​```python
# 这里是关键部分
code_here
​```

这段代码做了什么呢？
1. 第一行：[解释]
2. 第二行：[解释]
3. 第三行：[解释]

运行后输出：
​```
output_here
​```
```

## 文章开头设计

### 开头公式
1. **场景引入**（1句话）
2. **问题提出**（1个问题）
3. **价值说明**（1句话）
4. **内容预告**（可选）

### 开头示例
```
当你第一次学习编程时，一定会遇到"变量"这个概念。

什么是变量？为什么需要变量？

理解变量是编程的第一步，掌握它，你就能写出有用的程序。

本文将通过简单的例子，帮助你彻底理解变量的概念和用法。
```

## 文章结尾设计

### 结尾公式
1. **核心回顾**（2-3个要点）
2. **实践鼓励**（1句话）
3. **后续引导**（可选）

### 结尾示例
```
小结

本文我们学习了：
1. 变量是用来存储数据的容器
2. 变量可以改变值，常量不能
3. 使用变量让程序更灵活

现在，打开你的编辑器，动手试试吧！

下一节，我们将学习如何使用函数来组织代码。
```
